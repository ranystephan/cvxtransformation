The data was taken from the CRSP database through the following fields: 

PERMNO,date,SHRCD,TICKER,DISTCD,DIVAMT,PRC,VOL,RET,BID,ASK,CFACPR,CFACSHR


DATA PROCESSING:

The raw CRSP dataset underwent several processing stages.

Several columns were added to the dataset based on the following formulas:

*   `DIV_CASH`: This column was created to isolate cash dividends. Its value is equal to `DIVAMT` if the distribution code (`DISTCD`) starts with '12' (indicating an ordinary or special cash dividend), and 0 otherwise.
*   Split-Adjusted Columns: The price, volume, and bid/ask values were adjusted for stock splits using the cumulative factors provided by CRSP.
    *   `PRC_SPLIT_ADJ` = `PRC` / `CFACPR`
    *   `VOL_ADJ` = `VOL` / `CFACSHR`
    *   `BID_ADJ` = `BID` / `CFACPR`
    *   `ASK_ADJ` = `ASK` / `CFACPR`
*   `ADJ_CLOSE`: A total-return adjusted closing price was calculated. This was done by first creating a daily return factor: `(PRC_SPLIT_ADJ + DIV_CASH) / PRC_SPLIT_ADJ.shift(1)`. A cumulative product of these factors generated a total return index, which was then rescaled so its final value matched the final `PRC_SPLIT_ADJ`.
*   Spread Columns: Two metrics for the bid-ask spread were calculated.
    *   `SPREAD` = `ASK_ADJ` - `BID_ADJ`
    *   `REL_SPREAD` = `SPREAD` / ((`ASK_ADJ` + `BID_ADJ`) / 2)

Finally, the dataset was filtered to retain only the tickers belonging to the S&P 500 index, and the resulting data was pivoted and saved into separate files for prices, returns, spreads, and volumes.
